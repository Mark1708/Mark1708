# Шаблоны README репозиториев

Шаблоны для выравнивания публичных README в проектах Mark1708.

## Как выбрать шаблон

1. Начните с `README_BASE.ru.md` как общего публичного контракта.
2. Выберите один типовой шаблон по роли репозитория:

| Тип репозитория | Шаблон |
|---|---|
| CLI-инструмент / утилита разработчика | `README_TOOLS.ru.md` |
| Plugin host-приложения / OpenCode extension | `README_PLUGIN.ru.md` |
| Full-stack demo / приложение | `README_FULLSTACK.ru.md` |
| Backend-сервис / backend demo | `README_BACKEND.ru.md` |
| Frontend / статический сайт / web-приложение | `README_FRONTEND.ru.md` |
| Исследование / benchmark / ML-эксперимент | `README_RESEARCH.ru.md` |
| Библиотека / пакет | `README_LIBRARY.ru.md` |

## Общий обязательный контракт

Каждый публичный README должен содержать:

- one-line value proposition;
- единый dark/purple badge style;
- сводную таблицу со статусом, типом, основным стеком, быстрой проверкой и уровнем проверки;
- точные команды install/run/validation;
- ограничения или security caveats;
- явный словарь статусов.

## Уровни проверки

Используйте эти labels консистентно:

- **Документировано**: команда или утверждение есть только в README.
- **Основано на manifest**: утверждение совпадает с package, Gradle, Maven, Go, Python или другим manifest-файлом.
- **Подтверждено запуском**: команда запускалась и успешно прошла во время последнего обновления документации.

## Политика двух языков

Сначала стабилизируйте English README. Затем зеркальте тот же порядок секций в `README.ru.md`.

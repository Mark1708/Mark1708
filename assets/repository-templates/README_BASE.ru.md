## Проверка

| Область | Команда | Заметки |
|---|---|---|
| Сборка | `команда` | Документированная подсказка по сопровождению |
| Тесты | `команда` | Документированная подсказка по сопровождению |
| Lint/typecheck | `команда` | Документированная подсказка по сопровождению |

Уровни проверки:

- **Документировано**: команда или утверждение есть только в README.
- **Основано на manifest**: утверждение совпадает с package, Gradle, Maven, Go, Python или другим manifest-файлом.
- **Подтверждено запуском**: команда запускалась и успешно прошла во время последнего обновления документации.

## Ограничения / безопасность

- Укажите, это production-ready проект, демонстрация или исследовательский снимок.
- Документируйте caveats по auth, secrets, данным, benchmark или внешним сервисам.
- Держите примеры без секретов.

## Статус

Используйте один термин статуса консистентно:

- **Active**: текущая работа и поддержка ожидаются.
- **Maintained**: ожидаются исправления и обновления совместимости.
- **Finished demo**: завершенная демонстрация; продуктовая поддержка не подразумевается.
- **Research snapshot**: сохраненный эксперимент с заметками по воспроизводимости.
- **Archived**: активная поддержка не ожидается.
